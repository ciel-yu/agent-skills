# Information Sufficiency Criteria

Before generating the Product Spec, every condition below must be satisfied. If any item is unmet, keep probing — do not generate prematurely.

---

## Must Be Met

- [ ] **Product positioning is clear**: Can be stated in one plain sentence — what it is and what problem it solves
- [ ] **Core features are defined**: At least 3 features, each with a clear reason for existing
- [ ] **User flow is clear**: At least one complete path from opening the product to completing a task
- [ ] **Overall layout is explicit**: Number of columns known, regions and their contents defined
- [ ] **Widget conventions are clear**: Basic spec for inputs, buttons, and dropdowns is known
- [ ] **AI capability needs are explicit**: Which features use AI and what type of AI is known

---

## Judgment Principles

- User says "that's about it" but any item above is unmet: keep probing — no exceptions
- Ask one more round rather than generate something useless
- When information is sufficient, move forward — don't drag it out
